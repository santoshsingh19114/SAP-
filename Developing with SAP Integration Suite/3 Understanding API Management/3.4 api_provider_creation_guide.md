# 1. Overview
An **API Provider** in SAP Integration Suite is a configuration container that stores all backend system connection details. It is required before creating API Proxies in **API Management**.

Use it whenever you need to expose data from SAP or non-SAP backend systems.

---

# 2. What is an API Provider?
### **Definition:**
An API Provider is an object in SAP API Management that stores backend system connectivity details such as:
- Host
- Port
- Protocol (HTTP/HTTPS)
- Authentication
- Catalog service information
- Proxy or Cloud Connector settings

It acts as a **connection blueprint** for API creation.

---

# 3. Why Do We Use an API Provider?
To avoid entering backend details repeatedly.

### ‚úî Benefits:
- Centralizes backend configuration
- Simplifies API creation
- Ensures consistency
- Enables reachability tests
- Reduces duplication of effort

---

# 4. Where Is an API Provider Used?
API Providers are used inside:
### **Integration Suite ‚Üí API Management ‚Üí Configure ‚Üí API Providers**

You need it when:
- Designing API Proxies
- Prototyping services
- Fetching catalog services (OData/SOAP)
- Managing connections to SAP or non-SAP systems

---

# 5. When Do You Create an API Provider?
Create it when:
- Exposing backend APIs (OData/REST/SOAP)
- Connecting a new system to API Management
- Backend requires authentication, proxy settings, host/port info
- You want the same backend system reused across many APIs

---

# 6. Types of API Providers
SAP Integration Suite supports 5 main backend sources.

| No. | Source Type | Description |
|----|--------------|-------------|
| 1 | **Open Connectors** | 260+ prebuilt 3rd-party REST APIs |
| 2 | **On-Premise SAP Backend** (ECC, S/4HANA, PO/PI) | Access via Cloud Connector |
| 3 | **Cloud Integration (CPI/CI)** | OData/SOAP endpoints from CI artifacts |
| 4 | **Internet APIs** | Any public REST/SOAP API |
| 5 | **SAP Business Accelerator Hub** | Ready-to-consume SAP APIs |

---

# 7. Step-by-Step Procedure to Create an API Provider
You will perform these steps inside:
### **Integration Suite ‚Üí API Management ‚Üí Configure ‚Üí API Providers**

---

## **Step 1: Navigate to API Providers**
- Go to **Configure**
- Open **API Providers**
- Click **Create**

---

## **Step 2: Enter Overview Details**
Fill in the basic information:
- **Name** (e.g., `GWSAMPLE_BASIC_API_Provider`)
- **Description** (e.g., API Provider for ES5 Demo System)

---

## **Step 3: Configure Connection Details**
Fields include:
- Host  
- Protocol (HTTP/HTTPS)  
- Port  
- Proxy Type (Internet or On-Premise)  
- Cloud Connector Location ID (for on-prem)

### üìå **Mapping by Source Type:**
| Source Type | Backend | Mapping |
|-------------|---------|---------|
| Internet API | Public API | Type 4 |
| On-Prem Backend | SAP ECC/S4 | Type 2 |
| Open Connectors | External APIs | Type 1 |
| Cloud Integration | CPI-hosted APIs | Type 3 |

---

## **Step 4: Configure Catalog Service**
Enter catalog info such as:
- Path Prefix
- Service Collection URL
- Authentication Type
- Username / Password

### üìå SAP Tip:
For SAP S/4HANA ‚Üí the catalog service path is visible in:  
**Transaction: `/IWFND/MAINT_SERVICE`**

---

## **Step 5: Save and Test Connection**
After saving, click **Test Connection**.

### Possible valid results:
- **HTTP 200** ‚Üí Success
- **HTTP 404** ‚Üí Still valid (reachable system but wrong resource)

Both responses confirm backend reachability.

# 8. Create API Provider for ES5 Demo System
**ES5** (SAP Gateway Demo System) is commonly used for training and learning.

### üíº Scenario:
You want to expose **GWSAMPLE_BASIC** OData service via API Management.

### Steps:
1. Go to **Integration Suite ‚Üí API Management ‚Üí Configure**
2. Verify Virtual Host
3. Click **Create API Provider**
4. Enter ES5 host:  
   **https://sapes5.sapdevcenter.com**
5. Provide catalog service details
6. Enter ES5 username/password
7. Test connection ‚Üí Expect HTTP 200/404
